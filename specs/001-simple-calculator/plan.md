## Summary

Implement a stateless command-line calculator (`calc <operation> [operands]`) in TypeScript/Node.js that supports five operations ‚Äî `add`, `subtract`, `multiply`, `divide`, and `sin` ‚Äî plus correct error handling for invalid inputs, division by zero, and unsupported operations. The implementation follows the existing repo pattern of a thin CLI entry-point delegating to a pure library module, uses no new runtime dependencies, and is tested with the Node.js built-in `node:test` runner.

## Design Decisions (Phase 1 summary)

### Architecture

The feature is split into two layers to keep concerns separate and maximise testability:

1. **Library (`calculator.ts`)** ‚Äî Pure functions; no I/O; fully unit-testable without spawning a process.
   - `parseArgs(argv: string[]): CalcResult` ‚Äî validates operation and operands, returns typed result
   - `calculate(op: Operation, operands: number[]): CalcResult` ‚Äî performs arithmetic, returns typed result

2. **CLI (`cli.ts`)** ‚Äî Thin adapter; reads `process.argv.slice(2)`, calls library, routes output.
   - Success: `console.log(result.value)` ‚Üí `stdout`, exit `0`
   - Failure: `console.error('Error: ' + result.error)` ‚Üí `stderr`, exit `1`

### Key types

```typescript
type Operation = 'add' | 'subtract' | 'multiply' | 'divide' | 'sin';

type CalcResult =
  | { ok: true;  value: number }
  | { ok: false; error: string };
```

### Argument parsing (no third-party parser)

`process.argv.slice(2)` is destructured manually. Validation order:
1. No arguments ‚Üí usage error
2. Unrecognised operation ‚Üí unsupported-operation error  
3. Wrong operand count for operation ‚Üí operand-count error  
4. Non-numeric operand string ‚Üí invalid-input error  
5. `divide` with second operand `0` ‚Üí division-by-zero error  
6. All valid ‚Üí compute and return

### Error messages (exact strings ‚Äî tests must match)

| Case | Message |
|------|---------|
| No args | `Usage: calc <operation> [operands...]` |
| Unknown op | `Unsupported operation: "<op>". Supported: add, subtract, multiply, divide, sin` |
| Wrong count (binary) | `"<op>" requires 2 numeric operands` |
| Wrong count (unary) | `"sin" requires 1 numeric operand` |
| Non-numeric | `Invalid input: "<arg>" is not a number` |
| Divide by zero | `Division by zero` |

## Test Strategy

### Unit tests (`tests/calculator/calculator.test.ts`)

Test the library functions directly ‚Äî fast, isolated, no subprocess overhead.

| Suite | Cases |
|-------|-------|
| `add` | integer sum, decimal sum, negative operands |
| `subtract` | positive difference, result-is-zero, negative result |
| `multiply` | integer product, decimal product, multiply-by-zero |
| `divide` | exact quotient, decimal quotient, divide-by-zero error |
| `sin` | `sin(0) === 0`, `sin(œÄ/2) ‚âà 1`, negative input |
| Unsupported ops | `cos`, `tan`, `cot` ‚Üí error |
| Invalid inputs | non-numeric strings, empty string, missing operands |

### Integration tests (`tests/calculator/cli.test.ts`)

Spawn `cli.ts` as a subprocess via `node --import tsx/esm src/calculator/cli.ts` and assert `stdout`, `stderr`, and exit codes. Validates the full contract from `contracts/cli-contract.md`.

### Running tests

```bash
# Unit + integration
node --import tsx/esm --test tests/calculator/**/*.test.ts

# Watch mode
node --import tsx/esm --test --watch tests/calculator/**/*.test.ts
```

## Out of Scope (confirmed)

- `cos`, `tan`, `cot` operations ‚Äî explicitly rejected with error
- Degree input for `sin` ‚Äî radians only
- Compound / chained expressions
- Persistent calculation history
- Complex numbers
- REPL / interactive mode
- Compiled binary or `package.json` `bin` entry (internal dev tool only)
