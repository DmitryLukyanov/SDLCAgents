## Summary

Build a **zero-dependency TypeScript CLI calculator** that runs as an interactive REPL. The user types arithmetic expressions (with standard operator precedence, parentheses, unary minus, and the `sin` function in degrees) at a `> ` prompt; the calculator evaluates and prints the result, then re-prompts. The implementation uses a hand-written **recursive-descent parser** built entirely on Node.js built-in APIs — no npm packages required beyond the TypeScript dev toolchain already in the repo.

## Architecture

### Parser Strategy — Recursive Descent

All parsing and evaluation is done in a single-pass **recursive-descent parser with inline
evaluation** (no separate AST). This is the simplest correct approach for a fixed, small grammar
and requires no third-party library.

#### Grammar (EBNF)

```ebnf
expression  ::= term ( ( '+' | '-' ) term )*
term        ::= unary ( ( '*' | '/' ) unary )*
unary       ::= '-' unary | primary
primary     ::= NUMBER
              | 'sin' '(' expression ')'
              | '(' expression ')'
```

Production rules ensure:

- `*` and `/` bind tighter than `+` and `-` (standard precedence) — via `term` above `expression`
- Unary minus is right-associative and applies to any `primary` — via `unary` rule
- `sin` is handled as a named-function primary — keeps grammar extensible while limiting to exactly one function
- Parentheses correctly override precedence — via `primary` → `'(' expression ')'`

#### Token Types

```
NUMBER      floating-point literal (integer or decimal)
IDENT       identifier (only 'sin' is valid)
PLUS        +
MINUS       -
STAR        *
SLASH       /
LPAREN      (
RPAREN      )
EOF         end of input
```

#### sin — Degrees Conversion

```
sin(degrees) = Math.sin(degrees * Math.PI / 180)
```

Applied inside the evaluator at the `primary` production. No toggle or mode; always degrees.

### Number Formatting

```typescript
// 10 significant digits, strip trailing zeros, no trailing decimal point
function format(n: number): string {
  if (!isFinite(n)) return String(n);         // 'Infinity', '-Infinity', 'NaN'
  return parseFloat(n.toPrecision(10)).toString();
}
```

Behaviour table:

| Expression | Raw JS result | `toPrecision(10)` | After `parseFloat` | Displayed |
|---|---|---|---|---|
| `1 / 3` | `0.333…` | `"0.3333333333"` | `0.3333333333` | `0.3333333333` |
| `6 * 7` | `42` | `"42.00000000"` | `42` | `42` |
| `sin(90)` | `1` | `"1.000000000"` | `1` | `1` |
| `sin(180)` | `~1.22e-16` | `"1.224646799e-16"` | `1.224646799e-16` | `1.224646799e-16` |

### Error Handling

All errors are `throw new Error(message)` inside the lexer/parser/evaluator.
The REPL catches every error and prints `Error: <message>`, then re-displays `> `.
The process never exits on an evaluation error.

| Condition | Message |
|---|---|
| Division by zero | `Error: Division by zero` |
| Unknown character | `Error: Unexpected character '<ch>'` |
| Unknown identifier | `Error: Unknown function '<name>'` |
| Missing closing paren | `Error: Expected ')'` |
| Extra tokens after expression | `Error: Unexpected token '<tok>'` |
| Non-numeric `sin` argument | Caught as parse error before evaluation |

### REPL Lifecycle

```
startup
  └── readline.createInterface({ input: stdin, output: stdout })
       └── process.on('SIGINT') → rl.close() → process.exit(0)
            └── rl.question('> ', handler)
                 ├── trimmed === '' → re-prompt (FR-017)
                 ├── lower === 'exit' | 'quit' → rl.close() → process.exit(0)
                 ├── evaluate(line)
                 │    ├── success → print result → re-prompt
                 │    └── error   → print 'Error: ...' → re-prompt
                 └── loop
```
